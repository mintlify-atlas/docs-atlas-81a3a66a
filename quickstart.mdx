## Your First Command

marco-code uses a simple command-line interface. Pass your request with the `-p` flag:

```bash
marco-code -p "your prompt here"
```

The assistant will respond by calling tools (read, write, bash) as needed, then reply with text.

## Basic Example: Read a File

Let's ask marco-code to read a file:

```bash
marco-code -p "read the contents of README.md"
```

### What Happens

1. The LLM receives your prompt and the available tools (read, write, bash)
2. It decides to call the `read` tool with the file path
3. marco-code executes the tool and returns the file contents
4. The LLM summarizes the contents in natural language

You'll see tool execution in stderr:

```
  [tool] read({'file_path': 'README.md'})
```

Then the assistant's response in stdout:

```
The README.md file contains an introduction to marco-code...
```

## The Agent Loop in Action

marco-code demonstrates the core **agent loop** pattern used by AI coding assistants. Here's how it works:

<Steps>
  <Step title="User sends a prompt">
    You provide a task via the `-p` argument.
  </Step>

  <Step title="LLM decides what to do">
    The model receives:
    - Your prompt
    - Available tools (read, write, bash)
    - Working directory and platform info
    - Conversation history from previous iterations
  </Step>

  <Step title="Tools are executed">
    If the LLM calls tools, marco-code:
    - Executes each tool
    - Appends results to the conversation
    - Loops back to step 2
  </Step>

  <Step title="Final response">
    When the LLM has enough information, it responds with text and stops. The loop exits.
  </Step>
</Steps>

<Tip>
  The loop runs for a maximum of **10 iterations** to prevent infinite loops. You can see this in marco_code.py:52 (`MAX_ITERATIONS = 10`).
</Tip>

## Example: Write and Run Code

Let's see a more complex example that uses multiple tools:

```bash
marco-code -p "create a hello.py file that prints 'Hello from marco-code', then run it"
```

### Expected Output

Tool calls (stderr):
```
  [tool] write({'file_path': 'hello.py', 'content': "print('Hello from marco-code')"})
  [tool] bash({'command': 'python hello.py'})
```

Assistant response (stdout):
```
I've created hello.py and executed it. The output was:
Hello from marco-code

The script is now in your working directory.
```

## Available Tools

marco-code provides three tools to the LLM:

| Tool | Description | Example Usage |
|------|-------------|---------------|
| `read` | Read file contents | Read config files, source code |
| `write` | Write/create files | Generate code, create configs |
| `bash` | Execute shell commands | Run tests, install packages, git commands |

<Note>
  Bash commands have a **30-second timeout** (marco_code.py:52). Long-running commands will be terminated.
</Note>

## Working Directory Context

marco-code automatically tells the LLM:
- Your current working directory
- Your platform (OS and version)

This helps the LLM make platform-specific decisions (e.g., using correct path separators).

## Command-Line Arguments

marco-code has a minimal CLI:

```bash
marco-code -p "your prompt"
```

| Argument | Required | Description |
|----------|----------|-------------|
| `-p` | Yes | The prompt/task for the assistant |

<Warning>
  The `-p` argument is required. Running `marco-code` without it will show an error.
</Warning>

## Tips for Good Prompts

1. **Be specific**: Instead of "fix the code", try "fix the TypeError in main.py line 42"
2. **Provide context**: Mention relevant files or directories
3. **One task at a time**: Complex multi-step tasks work best when broken down
4. **Check the working directory**: Use `pwd` first if you're unsure where you are

## Example Prompts

Here are some real-world examples:

```bash
# Code analysis
marco-code -p "what does the calculate_metrics function do in utils.py?"

# Refactoring
marco-code -p "extract the database logic from main.py into a separate db.py module"

# Testing
marco-code -p "run pytest and show me any failing tests"

# File operations
marco-code -p "create a .gitignore file with common Python ignores"

# Shell commands
marco-code -p "show me the git status and recent commits"
```

## Viewing Tool Calls

Tool execution is logged to **stderr** so you can see what's happening:

```bash
# See tool calls in the terminal
marco-code -p "count lines in all Python files"

# Separate tool logs from output
marco-code -p "generate report" 2> tools.log

# Suppress tool logs
marco-code -p "say hello" 2> /dev/null
```
